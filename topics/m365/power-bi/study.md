# 준비

## 데이터 가공

쿼리는 데이터를 불러오고 가공하는 하나의 데이터 처리 단위

1. 데이터 쿼리 가져오기 (sales 2017, revenue, product)
2. 홈 탭에서 불필요한 상위 행 제거
3. 홈 탭에서 첫 행을 머리글로 사용
4. 보기 탭 열 품질
5. 홈 탭에서 빈행 제거
6. 비어있는 열의 필터에서 비어있음, NA 체크 풀기
7. 유니크 열 클릭 후 중복된 행 제거
8. 보기 탭의 열선택에서 불필요한 열 제거
9. 매출액 타입
10. 전체 열 선택 후 데이터 형식 검색 후 오류 찾기 (price, delivery_date_format2)
11. 다시 원래대로 변경 후 delivery_date_format2의 경우 로캘 사용 -> 영어(영국)선택
12. price 쿼리의 경우 NA 체크 풀기
13. 다시 변환 탭의 데이터 형식 검색
14. revenue, price 쿼리 고정 형식으로 변경
15. pormo_bin_1 홈 탭의 값 바꾸기로 verylow -> very low
16. sales를 숫자 형식으로 변경하기 위해 홈 탭의 값 바꾸기로 sales 없애고 형식 변경
17. store_city 데이터 가져오기
18. store size의 숫자를 뽑아오기 위해 변환 탭의 추출 -> 구분 기호 앞 텍스트에 공백 한 칸 입력
19. state - state abr - city를 분리하기 위해 변환 탭의 열 분할 -> 구분 기호 기준
20. 위도 경도도 19번과 같은 작업
21. 나눈 부분 열의 헤드 우클릭 -> 변환 -> 공백제거
22. product 쿼리 데이터 가져오기
23. 첫 행을 머리글로 사용
24. product 쿼리 헤드 좌클릭 -> 변환 -> 정리
25. length.. 부분, category 부분 열 분할
26. 홈 탭의 새 원본으로 sales 2018, 2019 가져오기
27. 홈 탭의 쿼리 추가는 선택된 쿼리에 다른 쿼리를 추가 임을 설명
28. 홈 탭의 쿼리를 새항목으로 추가
29. 첫 번째 테이블을 2018, 두 번째 테이블을 2019로

## 모델 연결

### 카테고리 별 매출 그래프

1. 누적세로막대형 차트를 편집기 데이터 창에 불러오기
2. prduct 테이블의 category를 x축으로 불러오기
3. sales 테이블의 revenue를 y축으로 불러오기 (빌드)
4. 모델보기에서 데이터 모델
5. 공통된 열 찾기
6. sales의 poduct id 와 porduct의 product id 연결

### state 별 매출 그래프

1. store의 state를 누적세로막대형으로 생성
2. sales의 revenue를 y축에 추가
3. sales의 store id와 stores의 store id 연결

### 도시 별 평균 가격

합계는 해당 도시의 모든 주문 price를 전부 더한 값
평균은 해당 도시의 주문 1건당 평균 가격

1. store의 city 를 누적세로막대형으로 생성
2. sales의 price를 y축으로 설정
3. y축의 price 합계를 평균으로 수정

## 그래프

### 빌드 패널

#### 범례

차트 안에서 데이터를 "한 번 더 나눠서" 보여주기 위한 기준

1. Category 별 매출 합계 그래프의 빌드에서 범례에 프로모션 타입 추가

#### 축소 다중 항목

한 차트를 여러 개의 작은 차트로 나눠서 보여주는 기능

1. 축소 다중 항목에 city 입력

### 서식 패널

1. 막대 그래프에 데이터 레이블 적용 해보기

### 파이 그래프

- 조각(각도/면적)을 비교하기 어렵기 때문에 값 비교 목적으로 사용 X
- 전체 대비 비율(Percentage of Total)을 보여줄 때 사용할 것
- 시선을 움직여가며 봐야하기 때문에 범례사용 X, 라벨을 사용할것

#### 주 별 매출

1. state와 revenue를 통해 원형차트 생성
2. 서식 패널의 세부 정보 레이블에서 범주와 총 퍼센트 선택
3. 빌드 패널의 자세히(데이터를 한 단계 더 세분화해서 보여주는 필드 영역)에 city 추가해 보면서 설명해주기 -> 다시 제거
